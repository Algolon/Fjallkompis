# PR evidence

Screenshots referenced from pull-request descriptions as visual evidence of
UI changes (PR bodies can only embed images by URL, so the evidence lives in
the branch/repo). One folder per PR, named `YYYY-MM-<topic>/`.

These are point-in-time captures for review, not living documentation — the
app itself and docs/DEVELOPMENT.md stay authoritative. Folders may be pruned
once a change is long merged and superseded.
